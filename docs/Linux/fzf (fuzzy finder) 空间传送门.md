# **fzf** (*fuzzy finder*) 空间传送门

在 Linux 开发中，我们经常要面对成千上万个文件、几百条历史命令，或者几十个正在运行的进程。**fzf** 的作用就是通过**模糊搜索**，让你在一瞬间从海量信息中抓取到你想要的那一个。

------

### 1. 核心魅力：什么是“模糊搜索”？

传统的搜索需要你完全拼对单词（比如 `grep "UserService"`），而 `fzf` 允许你乱序、缩写搜索。

比如你想找 `src/main/java/com/muzi/service/UserService.java`：

你只需要输入 `smjsUser`，`fzf` 就能帮你把它定位出来。

------

### 2. 安装与集成（关键一步）

`fzf` 不仅仅是一个工具，它更像是一个插件。

```
# 1. 安装
sudo apt install fzf

# 2. 启用键盘快捷键绑定（这一步是灵魂！）
# 不同的发行版路径不同，通常运行这个脚本即可：
source /usr/share/doc/fzf/examples/key-bindings.bash && source ~/.bashrc
```

------

### 3. 三大“神级”快捷键（开发必会）

安装并配置好后，你会解锁三个逆天的操作：

#### ① `CTRL + R`：历史命令“救命草”

这是最常用的场景。以前你可能要不停地按向上箭头，或者 `history | grep "mvn"`。

现在：按 `CTRL + R`，输入 `mvn run` 或者 `login`，`fzf` 会立即列出所有匹配的历史命令，回车即用。

#### ② `CTRL + T`：快速找文件

在终端想打开某个深层目录的文件？

输入 `vim ` 然后按 `CTRL + T`，直接模糊搜索文件名，选中后文件名会自动补全到 `vim` 后面。

#### ③ `ALT + C`：快速切目录

想去某个文件夹？按 `ALT + C`，输入文件夹名的几个字母，瞬间闪现过去。

------

### 4. 配合你的 Spring Boot 开发流

#### 场景一：快速查看日志

如果你的日志文件很多，或者你想在控制台输出中找特定错误：

```
cat spring.log | fzf
```

你可以在弹出的界面里实时过滤 `ERROR` 或 `Exception`。

#### 场景二：杀死占用的进程

Spring Boot 端口被占了？不用再 `ps -ef | grep java` 然后手动复制 PID：

```
ps -ef | fzf | awk '{print $2}' | xargs kill -9
```

直接在 `fzf` 列表里选那个 Java 进程，回车直接杀掉。

#### 场景三：配合 `vim`

如果你在用 Vim 改代码，在终端输入：

```
vim $(fzf)
```

这会先让你选文件，选完直接进入编辑。

------

### 5. 进阶：预览功能 (Preview)

这是 `fzf` 最酷炫的地方。你可以配置它在搜索时，侧边栏直接显示文件内容：

```
# 搜索文件时，右侧显示前100行内容
fzf --preview 'cat {} | head -100'
```

*提示：如果安装了 `bat`（带高亮的 cat），预览效果堪比 IDE。*
