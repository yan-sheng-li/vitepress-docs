# Micro 现代化的终端编辑器

------

### 1. 为什么 Micro 是 Nano 的终极替代品？

- **真正的鼠标支持**：你可以直接用鼠标点击定位、滚动轮翻页、甚至**拖拽选中**代码（这在普通终端编辑器里是奢侈品）。
- **现代快捷键**：它支持 `Ctrl+C` (复制)、`Ctrl+V` (粘贴)、`Ctrl+S` (保存)、`Ctrl+F` (搜索)。你不需要学习像 Vim 那种反直觉的指令。
- **原生高亮**：默认支持 75+ 种语言（Java, SQL, YAML, JSON 等），效果非常出色。
- **内置终端**：按 `Ctrl+E` 然后输入 `term`，你可以在编辑器里直接开一个分屏终端跑 Maven 命令。

------

### 2. 安装（极简安装）

依然推荐通过包管理器或脚本安装，它是一个单一的 Go 二进制文件，非常纯净：

```
sudo apt install micro
```

------

### 3. 核心功能与你的“格式化”诉求

Micro 解决你“格式化”疑惑的姿势非常优雅：

#### ① 强大的插件系统

Micro 支持插件，你可以安装格式化插件。按 `Ctrl+E` 进入命令模式，输入：

```
plugin install fmt
```

它会自动调用系统中的格式化工具（如 `gofmt` 或其他）。

#### ② 联动外部工具（万能格式化）

如果你想用我们刚才聊的 `jq` 格式化当前编辑的 JSON，在 Micro 里只需要：

1. 按 `Ctrl+E`。
2. 输入 `run jq .` (这会把当前文件内容喂给 jq 处理)。
3. 或者直接配置一个快捷键。

------

### 4. 实战提效技巧

#### 分屏操作（全栈开发刚需）

在开发 Spring Boot 时，你可能想左边看 `Service`，右边看 `Controller`：

- `Ctrl+E` 键入 `vsplit UserService.java`：垂直分屏。
- `Ctrl+E` 键入 `hsplit log.txt`：水平分屏。
- `Alt+,` 和 `Alt+.` 在分屏间来回穿梭。

#### 多光标编辑

这是 Micro 媲美 VS Code 的地方：

- 按住 `Ctrl` 点击不同行，或者按 `Alt-n` 选择下一个匹配项。
- 你可以同时修改多行代码，这在批量修改 `application.yml` 配置时非常爽。

## 附录

### 1. 快捷键：回归人类直觉

Micro 最大的卖点就是它支持你在 Windows 或浏览器里已经养成的习惯：

- **保存**：`Ctrl + S`
- **复制 / 剪切 / 粘贴**：`Ctrl + C` / `Ctrl + X` / `Ctrl + V`
- **撤销 / 重做**：`Ctrl + Z` / `Ctrl + Y`
- **查找**：`Ctrl + F`
- **全选**：`Ctrl + A`
- **退出**：`Ctrl + Q`（如果文件没保存，它会温柔地询问你）

------

### 2. 核心命门：命令模式 (`Ctrl + E`)

就像 IDEA 的 `Shift + Shift` 或 VS Code 的 `Ctrl + Shift + P`，在 Micro 里，几乎所有高级功能都通过按 **`Ctrl + E`** 调出底部的命令行。

常用命令：

- **`help`**：查看自带的详细教程。
- **`set [option] [value]`**：修改配置。例如 `set colorscheme solarized`。
- **`plugin install [name]`**：安装插件。
- **`vsplit` / `hsplit`**：垂直或水平分屏编辑。

------

### 3. 分屏开发技巧

开发 Spring Boot 时，你经常需要对照两个文件。Micro 的分屏非常直观：

1. **开启分屏**：按 `Ctrl + E` 输入 `vsplit UserService.java`。
2. **切换焦点**：按 `Alt + ,` 或 `Alt + .` 在左右窗口间跳转。
3. **关闭窗口**：直接在那个窗口按 `Ctrl + Q`。

------

### 4. 鼠标操作（Nano 做不到的事）

- **直接点击**：鼠标点哪里，光标就闪在哪里。
- **拖拽选中**：用鼠标划选一段代码，直接按 `Ctrl + C` 复制。
- **滚动滚轮**：在终端里也能像看网页一样快速翻阅代码。

------

### 5. 格式化与自动提示

- **语法高亮**：它会自动识别你的 `.java`、`.yml`、`.sql` 扩展名。如果没识别出来，按 `Ctrl + E` 输入 `set filetype java`。
- **自动缩进**：它比 Nano 聪明得多，回车会自动对齐上一行的缩进级别。
- **行号显示**：默认开启，方便你定位 Spring Boot 报错堆栈里的具体行数。

------

### 6. 配置持久化

Micro 的配置文件位于 `~/.config/micro/settings.json`。你也可以通过命令模式直接改并保存。

如果你习惯用鼠标滚轮，但发现有时候会进入“选中模式”，可以尝试：

Bash

```
# 在 micro 内部按 Ctrl+E 执行
set mouse true
```
