

# WSL 科学上网完整配置流程（稳定版）

适用于 **Windows Subsystem for Linux + Windows 代理（Clash / V2Ray）**

------

## 🟢 Step 1：确认 WSL 网关（Windows 入口）

在 WSL 执行：

```bash
ip route | grep default
```

得到类似：

```
default via 172.18.176.1 dev eth0
```

👉 记住这个 IP（Windows 网关）

------

## 🟢 Step 2：确认代理端口连通

测试端口：

```bash
nc -vz 172.18.176.1 7890
```

看到：

```
succeeded
```

✔ 说明 WSL 能访问 Windows 代理端口

------

## 🟢 Step 3：设置代理环境变量

```bash
export http_proxy=http://172.18.176.1:7890
export https_proxy=http://172.18.176.1:7890
```

（可选增强）

```bash
export all_proxy=http://172.18.176.1:7890
```

------

## 🟢 Step 4：测试是否生效

```bash
curl https://www.google.com
```

看到 HTML 页面内容 = 成功

------

## 🟢 Step 5：永久生效（推荐）

写入 bash 配置：

```bash
nano ~/.bashrc
```

追加：

```bash
export http_proxy=http://172.18.176.1:7890
export https_proxy=http://172.18.176.1:7890
export all_proxy=http://172.18.176.1:7890
```

生效：

```bash
source ~/.bashrc
```

------

## ⚠️ Windows 端必须满足

在你的代理软件（Clash / V2Ray）：

✔ 开启 **Allow LAN（局域网访问）**
✔ 监听端口 7890（HTTP）

------

## 🧠 核心原理（必须理解）

```text
WSL → Windows 网关 (172.18.176.1)
     → Windows 代理端口 (7890)
     → 科学上网出口
```

------

## 🚀 一句话总结

👉 WSL 不直接继承 Windows 代理
👉 必须手动把流量指向 Windows 网关 + 代理端口
👉 成功标志：`nc 通 + curl 出 HTMLwsl配置
